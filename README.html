How to use
Local browser: Double-click index.html to open in Safari, Chrome, or Firefox. All functionality works offline — the application loads D3.js from a CDN (requires internet for first load; fonts also require internet). No server, no installation.

Publishing: Upload the folder to any static web host (GitHub Pages, Netlify, Cloudflare Pages, S3 static hosting). No backend required. If deploying to a subdirectory, all file references are relative and will resolve correctly.

Updating data: Open index.html in a text editor and find the const FACILITIES = [ array. Each facility is a JavaScript object — add entries following the same field structure. The application will automatically include new entries in all views. The data_manifest.csv should be updated in parallel to keep the dataset synchronized.

Peer review: The methodology document (methodology.html) contains the full reproducibility guide in Section 9, including step-by-step instructions for obtaining primary sources and validating each estimate tier.

Key findings summary
Most intensive per acre: Google The Dalles (7.02 AF/ac/yr) withdraws more water per acre than any irrigated crop in the USDA database except semiconductor fabrication facilities. It withdraws 5.3× more water than falls as precipitation on its own campus footprint and supplies 1/3 of the City of The Dalles' total water for a population of ~16,200 people.

Best-in-class: Microsoft Quincy (post-2021 QWRU closed-loop) and Switch Citadel (air-cooled) demonstrate that geography + technology can bring water consumption to near zero regardless of scale. Google's Hamina, Finland campus (seawater cooling) eliminates freshwater dependency entirely.

Counterintuitive residential finding: Dense urban residential land (apartments, Q5) uses only ~0.6 AF/acre/year — less than semi-rural residential (~1.8 AF/ac) and less than most data centers. High population density reduces water intensity per acre by eliminating landscaping and enabling efficient shared infrastructure.

Disclosure gap: Only Google provides site-level water data proactively. Most large operators' water footprints are unknown or estimable only through public records requests to local utilities — a significant accountability gap given the scale of consumption.
